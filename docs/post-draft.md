# Скриншоты для AI-чатов: маленькая утилита, которая сильно ускорила мой vibe-coding

> 5 минут чтения, 1 минута установки. Windows-only, Python, MIT.

## Проблема

Я кожу через терминал (Claude Code), и каждый день скидываю моделям десятки скриншотов: ошибки в консоли, кусочки UI, фрагменты кода в чужом репо, обрывки документации. Стандартный workflow на Windows такой:

1. `Win+Shift+S` — выделить область, картинка в буфере
2. Куда вставить, чтобы Claude её прочёл? В терминал картинку не вставишь
3. Открыть Paint → `Ctrl+V` → File → Save As → выбрать папку → имя → ОК
4. Скопировать путь к сохранённому файлу
5. Вставить путь в терминал

Каждый скриншот — 5 шагов. За день это сотни лишних движений.

## Что хотелось

Утилита, которая после захвата области:
- кладёт картинку в `~/Pictures/Screenshots` сама
- кладёт **путь к файлу** в буфер обмена
- так чтобы `Ctrl+V` в терминале мгновенно дал путь, и Claude автоматически прочитал картинку

Плюс — если в скриншоте просто длинный текст ошибки, было бы удобно одним хоткеем превратить картинку **в текст** (через OCR) и положить в буфер уже текст. Меньше токенов, и модель не путает `l` с `1`.

## Что получилось

`screenshot-to-terminal` — ~500 строк Python, иконка в трее, четыре хоткея:

| Hotkey               | Что делает                                                       |
|----------------------|------------------------------------------------------------------|
| `Ctrl+Alt+S`         | Сохранить картинку из буфера через диалог (имя файла выделено)  |
| `Ctrl+Alt+Shift+S`   | То же, но мгновенно в `~/Pictures/Screenshots`                  |
| `Ctrl+Alt+A`         | Захват области через свой spotlight-overlay → сразу в файл      |
| `Ctrl+Alt+D`         | OCR картинки из буфера → распознанный текст в буфер             |

Каждое сохранение кладёт в буфер **и картинку, и путь одновременно** (Windows clipboard поддерживает несколько форматов). Терминал увидит путь, Paint — картинку. Никаких компромиссов.

## Несколько мелочей, которые радуют

- В диалоге сохранения имя файла автоматически выделено — можно сразу начать печатать своё, не нужно выделять руками
- Префикс имени по проекту: настроил `bugfix-auth` → файлы стали `bugfix-auth_2026-05-18_14-22-01.jpeg`, потом удобно разгребать
- Авторесайз больших скринов до 1920px по длинной стороне (экономит токены, не теряет читаемости)
- OCR через **Windows.Media.Ocr** — встроенный, без интернета, без установки tesseract. Работает на 60+ языках (нужен языковой пакет)
- Запоминает последнюю папку сохранения
- Защита от двойного запуска через named mutex
- Иконка в трее с шпаргалкой по хоткеям и переключателями

## Почему не ShareX?

ShareX — гигант с 100+ фичами и steep learning curve. Я хотел утилиту с одним фокусом: положить путь и картинку в буфер так, чтобы вставка в AI-чат работала одним нажатием. Это получилось.

Если ShareX вам подходит — отлично. Эта утилита для тех, кто хочет минимум.

## Стек

Python 3.13, `pywin32`, `pystray`, `keyboard`, `Pillow`, `winsdk`. Один `.pyw`-файл. Можно собрать в `.exe` через PyInstaller (есть `build.bat`).
